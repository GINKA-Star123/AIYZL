# AIYZL 使用手册

## 1. Python 安装

### 1.1 下载 Python

1. 访问 [Python 官方网站](https://www.python.org/downloads/)
2. 下载 Python 3.13 或更高版本的安装包
3. 选择与您操作系统匹配的版本（Windows、macOS 或 Linux）

### 1.2 安装 Python

#### Windows 系统

1. 运行下载的安装程序
2. 勾选 "Add Python to PATH" 选项
3. 点击 "Install Now" 完成安装
4. 安装完成后，打开命令提示符，输入 `python --version` 验证安装成功

## 3. 虚拟环境创建与激活

### 3.1 创建虚拟环境

```bash
# Windows
python -m venv .venv

```

### 3.2 激活虚拟环境

```bash
# Windows
.venv\Scripts\activate

```

激活后，命令提示符前会显示 `(.venv)` 表示虚拟环境已激活。

## 4. 依赖安装

1. 确保虚拟环境已激活
2. 安装项目依赖

   ```bash
   pip install -r requirements.txt
   ```

   这将安装所有必要的依赖包，包括：(注：torch模块请根据GPU cuda版本去官网进行安装)
   - 语音识别相关：faster-whisper、webrtcvad、pyaudio
   - 自然语言处理：sentence-transformers、transformers、langchain
   - 深度学习：torch
   - 向量检索：faiss-cpu
   - GUI 界面：PyQt5
   - Live2D 渲染：live2d-py
   - 网络通信：websockets、requests、aiohttp
   - 其他工具库

## 5. 环境配置

### 5.1 API 密钥配置 （不用管，直接用我的API就行）

1. 打开 `.env` 文件
2. 添加您的 SiliconFlow API 密钥：
   ```
   SILICONFLOW_API=your_api_key_here
   ```


## 6. 系统启动

1. 确保虚拟环境已激活
2. 启动系统
   ```bash
   python main.py
   ```

3. 系统启动后，会显示以下信息：
   ```
   ✅ 系统已启动
      ASR 监听中
      💬 B站弹幕监听中
      Ctrl+C 退出
   ```

4. TTS启动，请前往data/models/GPTSOVITS 中运行 api_v2.bat文件

## 7. 功能使用

### 7.1 语音交互

1. 系统启动后，会自动开始监听麦克风
2. 对着麦克风说话，系统会自动识别您的语音
3. 系统会生成智能回复并通过语音合成播放出来
4. 说话结束后，系统会在 0.8 秒后自动处理您的语音

### 7.2 Live2D 模型展示

1. 系统启动后，会在屏幕上显示 Live2D 角色
2. 您可以通过鼠标拖动窗口位置
3. 角色会根据系统的输出进行相应的动画展示

## 8. 系统停止

1. 在命令行窗口中按 `Ctrl+C`
2. 系统会安全退出，释放所有资源

## 9. 故障排除

### 9.1 语音识别失败

- **问题**：系统无法识别语音
- **解决方案**：
  1. 检查麦克风是否正常工作
  2. 确保环境安静，减少背景噪音
  3. 检查 CUDA 环境是否正确配置（如果使用 GPU 加速）
  4. 首次运行时，系统会下载 Faster Whisper 模型，确保网络连接稳定

### 9.2 语音合成失败

- **问题**：系统无法合成语音
- **解决方案**：
  1. 确保 GPTSOVITS 服务正常运行
  2. 检查网络连接是否正常
  3. 检查 `service/TTS/GPTSOVITS.py` 中的服务地址是否正确

### 9.3 Live2D 模型不显示

- **问题**：Live2D 角色不显示
- **解决方案**：
  1. 检查 `Resources` 目录下的模型文件是否存在
  2. 确保 PyQt5 正确安装
  3. 检查 `service/live2d/live2dDisplay.py` 中的模型路径是否正确



### 9.4 内存溢出

- **问题**：系统出现内存溢出错误
- **解决方案**：
  1. 确保您的电脑有足够的内存（推荐 8GB 以上）
  2. 系统会自动清理 GPU 显存，无需手动操作（显存至少8G）
  3. 如果问题持续，尝试减少同时运行的服务数量

## 10. 性能优化

### 10.1 GPU 加速

- 确保 CUDA 环境正确配置，以加速语音识别
- 系统会自动检测 GPU 并使用 GPU 加速

### 10.2 内存管理

- 系统会定期清理 GPU 显存，避免内存溢出
- 音频缓冲区有大小限制，防止内存占用过高

### 10.3 音频处理

- 优化了音频缓冲区管理，提高识别准确率
- 针对中文语音进行了优化，提高识别质量

## 11. 安全注意事项

- 环境变量中的 API 密钥不要泄露（不要泄露我的API谢谢）
- 不要在公共场合使用系统的语音识别功能
- 定期清理 `knowledge/Personal/memory.json` 文件，避免敏感信息积累（这里你看个人）
- 不要将包含 API 密钥的配置文件提交到版本控制系统

## 12. 常见问题

### 12.1 首次运行时下载模型很慢

- **解决方案**：请耐心等待，模型下载完成后会自动开始使用（
- **建议**：在网络连接稳定的环境下首次运行

### 12.2 系统占用资源过高

- **解决方案**：
  1. 关闭不必要的后台程序
  2. 减少同时运行的服务数量
  3. 调整 `service/ASR/ASR.py` 中的 `compute_type` 参数为 "int8"

### 12.3 语音识别准确率不高

- **解决方案**：
  1. 确保环境安静，减少背景噪音
  2. 说话清晰，语速适中
  3. 调整 `service/ASR/ASR.py` 中的 VAD 参数
